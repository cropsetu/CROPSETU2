# Farm Detail

> **Tab:** My Farm · **Stack:** `MyFarmStack` (also re-registered in `AIStack` for deep-linking) · **Route name:** `FarmDetail` · **File:** [FarmDetailScreen.js](../../../frontend/src/screens/FarmProfile/FarmDetailScreen.js)

## Purpose
The single-farm overview ("one plot, all its seasons"). For one selected farm it gathers, in a single scroll: a hero stat strip (area, soil type, water source) with an "active farm" toggle, a row of glance cards (soil health / weather / nearby mandi), a season financial summary (cost of cultivation, revenue, net P/L, yield/acre), AI insights from FarmMind, the list of crop cycles on this plot with their stage timeline, and entry points to edit the farm or start a new crop cycle. It is the hub a farmer lands on after picking a farm, and the launch pad into each crop cycle's [Crop Cycle Detail](06-CropCycleDetailPage.md) screen.

## Where it sits / how you reach it
- **Reached from:**
  - [FarmListScreen](02-FarmListPage.md) — tapping a farm card → `navigate('FarmDetail', { farmId })`.
  - [MyFarmHomeScreen](01-MyFarmHomePage.md) — the "Manage farms" / plot chips path resolves through the list into this detail.
  - Deep links from the AI tab (the screen is re-registered in `AIStack`) so "open my farm" round-trips here.
- **Navigates to:**
  - [FarmAddEditScreen](04-FarmAddEditPage.md) via the header edit icon (`navigate('FarmAddEdit', { farm })`) — Stage 0 plot header edit.
  - [CropCycleCreateScreen](05-CropCyclePlanPage.md) (the guided **Crop Plan** — a single-scroll form, not a multi-step wizard) via "Start a cycle" / "Add cycle" (`navigate('CropCycleCreate', { farmId })`).
  - [CropCycleDetailScreen](06-CropCycleDetailPage.md) via any cycle row (`navigate('CropCycleDetail', { cycleId })`).
  - The AI tab: `AIAssistant` → `AIChat` (FarmMind, seeded with farm context), `SoilHealth`, `InputCalculator`, `Market` — from the insight "Why this?" actions and the "Ask FarmMind" 2×2 grid.
  - *(Redesigned)* [GrowthStoryScreen](09-GrowthStoryPage.md) for a cycle, via a "View Growth Story" secondary CTA on each cycle row / the active-cycle hero.
- **Route params in:** `farmId` (required) — the farm to load. Everything else is fetched.

## How it works
- On mount and on **every focus** ([`useFocusEffect`](../../../frontend/src/screens/FarmProfile/FarmDetailScreen.js)) the screen calls `farmApi.getFarm(farmId)` and stores the whole farm object (which embeds `soilReports[]` and `cropCycles[]`). Re-loading on focus means a cycle created in the wizard, or a stage advanced in the detail screen, shows up without a manual pull. Pull-to-refresh re-runs the same load.
- **Derived values:** `isActive = farm.id === activeFarmId` (from `MultiFarmContext`); `soil = farm.soilReports[0]`; `cycles = farm.cropCycles`; `insights = computeInsights(farm, soil, cycles)` — a local heuristic set (see Notes). `farmName` falls back `farmName → farmAlias → "Farm {farmNumber}"`; `location` joins village/taluka/district.
- **Active-farm toggle:** when the farm is not active, a "Set as active farm" pill calls `switchActiveFarm(farm.id)`; when active it shows "Active · FarmMind uses this data". The active farm is the one the AI tab and dashboard treat as default.
- **Crop cycles:** each `CycleRow` shows the [CropIcon](../../../frontend/src/components/CropIcons.js), crop + variety, area/season/year, and a [`StageTimelineBar`](../../../frontend/src/screens/FarmProfile/ui/StageTimelineBar.js). **DAS** (Days After Sowing) is computed inline as `sowingDate ? max(0, floor((now − sowingDate)/86400000)) : null` and rendered as "Day N after sowing". The displayed stage is the authoritative `cycle.growthStage` (never a DAS-derived guess).
- **Soil glance:** when a soil report exists it renders five `RadialGauge`s (pH, N, P, K, OC) colored by `soilColor(rating)` (green = high/adequate/optimum/normal, red = low/deficient, amber otherwise); otherwise an empty card nudges the farmer to upload a Soil Health Card.
- **Loading / not-found:** a centered spinner while loading; a "Farm not found" empty state (leaf icon + "Go back") when the fetch fails or returns nothing.
- **KhetAI re-theme (token values ship now; per-screen layout chrome still pending).** This screen imports `COSMIC/CR/CS/CT/GLOW` from [cosmicTheme.js](../../../frontend/src/screens/FarmProfile/theme/cosmicTheme.js), whose token *values* are already remapped to the KhetAI/Login design system: deep forest-green primary `#005F21`, **gold** accent `#E0AF3B` (formerly orange), forest-tinted text, KhetAI warm green-white surfaces, and `GLOW` as a soft green KSHADOW (no neon). `CT.family` is **Plus Jakarta Sans** for body plus **Fraunces** for display (`CT.family.display/displaySemi/displayItalic`), and every `Inter_*` literal in this file was swapped 1:1 to `PlusJakartaSans_*`. So FarmDetail **already renders forest-green/gold + Plus Jakarta today** — it is *not* the old blue/amber/Inter look. What is *not yet* applied here is the per-screen **layout chrome** seen on the Login reference: Fraunces serif section/number titles, the italic coloured second line (e.g. "Your wheat,\n*growing strong.*"), and 56px gradient icon-square headers. `CosmicHeader`'s title is already Fraunces serif, but FarmDetail's hero stats, section labels and big numbers still use the Plus Jakarta weights, not serif. The weather/nearby-mandi glance cards, the server-backed insights feed, and the financial-summary card described below as "redesigned scope" remain **unwired on this screen** (see Notes).

## Services, APIs & data
- **API (all [farmApi.js](../../../frontend/src/services/farmApi.js)):**
  - `getFarm(farmId)` → `GET /farms/:id` — the farm with embedded `soilReports[]` and `cropCycles[]` (**the only call `FarmDetailScreen.js` wires today**).
  - `getFarmInsights(farmId, { limit, type })` → `GET /farms/:id/insights` — server-side advisory feed (the intended replacement for the local `computeInsights`). **Defined in farmApi and already consumed by [MyFarmHomeScreen](01-MyFarmHomePage.md), but NOT called from this screen yet.**
  - `getFarmFinancialSummary(farmId, { season, year })` → `GET /farms/:id/financial-summary` — would back the season financial-summary card. **Defined in farmApi but NOT imported or called by `FarmDetailScreen.js`** (re-verified against the shipped file).
  - `switchActiveFarm(farmId)` (via `MultiFarmContext`, backed by `setActiveFarm` → `POST /farms/active`).
- **Backend route/service:** `backend/src/routes/farm.routes.js` for `/farms/:id`, `/insights`, `/financial-summary`, `/active`. Per-cycle financials are computed by `computeFinancials()` in [cropCycle.service.js](../../../backend/src/services/cropCycle.service.js) (cost of cultivation, cost/acre, revenue, net P/L, yield/acre, B:C ratio); the farm summary aggregates across this plot's cycles.
- **Data model:** each cycle is a `FarmCropCycle` ([schema.prisma](../../../backend/prisma/schema.prisma)) — `season`, `year`, `cropName`, `variety`, `areaAllocatedAcres`, `sowingDate`, `growthStage` (PLANNING → LAND_PREP → SOWING → VEGETATIVE → FLOWERING → FRUITING → MATURITY → HARVESTED), seed fields, and the JSON event arrays (`irrigationLogs[]`, `fertilizersUsed[]`, `pesticidesUsed[]`, `observedEvents[]`, `activities[]`, `laborLogs[]`, `expenseLogs[]`, `incomeLogs[]`).
  - *Note — stage label mapping:* the enum value `FRUITING` is horticulture wording. For most Indian field crops the UI should read it **per crop category**: "Grain filling" for cereals (wheat/rice/maize), "Pod development" for pulses & oilseeds (gram/tur/soybean/mustard/groundnut), "Boll development" for cotton, and "Fruit set/development" for veg & fruit. The stored enum stays one value; only the displayed label varies by `cropCategory` (which is itself an unwired schema column — see gaps).
- **State / context:** `useMultiFarm()` (`activeFarmId`, `switchActiveFarm`), `useLanguage()` (`t`); local `useState` for `farm`, `loading`, `refreshing`. `Haptics` on set-active and prediction taps.
- **Local / static data:** `computeInsights`, `soilColor`, `STAGES` order via `StageTimelineBar`; theme tokens (`COSMIC`, `CR`, `CS`, `CT`, `GLOW`) from [cosmicTheme.js](../../../frontend/src/screens/FarmProfile/theme/cosmicTheme.js); `CropIcon`, `RadialGauge`.

## Languages / i18n
- **Today:** the screen mixes hard-coded English strings ("Today's insights", "Active crops", "Soil health", "Ask FarmMind", "Start a crop cycle", insight bodies) with a few `farmProfile.*` keys (`farmProfile.loadError`, `farmProfile.notFound`, `farmProfile.askFarmMind`, `farmProfile.bestCrop`, `farmProfile.seedQty`, `farmProfile.income`, plus the `farmProfile.*` PredCard subtitles) and `login.error`.
- **(Redesigned)** All My Farm copy moves to the **`myFarm.v2.*`** namespace (e.g. section titles, financial-summary labels, glance-card labels, insight templates), with bilingual crop/variety from `cropName`/`cropNameMr`/`cropNameHi`. A glass language-pill in the header toggles language; advisory text from `getFarmInsights` can arrive localized from the server.

## Notes, edge cases & gaps
- **Local insight heuristics today.** `computeInsights` is an explicitly-labelled placeholder ("until v2 context-aware backend lands") firing on `soil.nitrogenRating === 'low'`, acidic pH, `RAINFED` farms, missing soil report, and zero cycles. The redesigned screen swaps this for `getFarmInsights`; document the heuristic version as current behaviour and the server feed as intended.
- **Not yet wired on THIS screen (redesigned scope):** the **weather** and **nearby mandi** glance cards, the **season financial summary** card, the server-backed insights feed, and a **Growth Story** CTA. The theme remap itself is done — FarmDetail already imports the legacy `cosmicTheme` tokens **now remapped to KhetAI forest-green/gold + Plus Jakarta** (so this is no longer a gap; only the per-screen serif/gradient *layout* chrome is). On the data path, **`getFarm(farmId)` is the only API this screen calls**, and `computeInsights` (local heuristic) is still the live insight source. `getFarmFinancialSummary(farmId, { season, year })` and `getFarmInsights(farmId)` both exist in [farmApi.js](../../../frontend/src/services/farmApi.js) but are **NOT imported or called by `FarmDetailScreen.js`** — re-verified against the shipped file. None of the `FarmProfile/components/` cards are imported here either (see catalogue below).
- **`FarmProfile/components/` glance cards — what exists vs. what's wired.** The redesigned cards are built and live in [`FarmProfile/components/`](../../../frontend/src/screens/FarmProfile/components/), but **none of them are imported by `FarmDetailScreen.js`**. They all read the remapped `COSMIC`/KhetAI tokens, so wiring them in is a layout task, not a re-theme. Status today:

  When this screen's redesign lands, the inline soil row, `computeInsights`, and the (currently absent) weather/mandi/financial blocks should be replaced by these five cards. Until then, treat them as ready-but-shelved.
- **Authoritative vs. derived stage.** The `StageTimelineBar` highlights `cycle.growthStage` (set manually via `advanceStage`). DAS is shown for context only and must never override the stored stage — relevant when a cycle is paused or off the typical calendar.
- **Pre-sowing cycles.** A cycle created via the [Crop Plan](05-CropCyclePlanPage.md) form can exist at `PLANNING`/`LAND_PREP`/`SOWING` with `sowingDate` still null → DAS is null and the timeline bar omits the "Day N" line. (`sowingDate` is now stamped by the Sowing logger at `advanceStage('SOWING')`, so DAS starts ticking from sowing.) The financial summary for such a cycle is mostly zeros (only pre-seeding spend, e.g. seed and land-prep, logged so far).
- **Soil card depth.** `soilReports[0]` is the latest report only; there is no in-screen history switcher here (that lives in the Soil hub). Partial reports render fewer gauges where values are null.
- **The unused `SoilBadge` component** is defined but not rendered (the screen uses `RadialGauge`s); harmless dead code to be aware of when editing.
- **Schema columns no screen writes yet.** Several `FarmCropCycle` columns are defined in Prisma but are **not captured by any screen** (including the [Crop Plan](05-CropCyclePlanPage.md) creator and the [activity loggers](08-ActivityLoggers.md)): `seedReceiptUrl`, `expectedHarvestDate`, `cropCategory`, `cropNameMr`/`cropNameHi`, `seedCostPerKgInr`, and `seedPurchaseDate`. They stay schema-only until a screen wires them — relevant because the bilingual crop names and the per-category stage label (above) and acres↔₹/quintal logic all want `cropCategory`/`cropNameMr`/`cropNameHi` to be populated.
- **Multi-plot future.** The header comment notes plot-level tabs are deferred until the Plot/ActivityLog tables ship; today one "farm" maps to one plot and the screen scrolls top-to-bottom.
- **India-specific.** Area is **acres-only** on this screen *and* in the [Crop Plan](05-CropCyclePlanPage.md) screen — the area step shows a fixed "ACRES" pill with a "Regional units (bigha, guntha) coming soon" note; there is **no** guntha/bigha/kanal/ha auto-conversion yet (bigha in particular is region-variable, so a state-aware conversion is needed). The future financial-summary and mandi cards are rupee-denominated and should resolve to the farmer's nearest APMC for the active crop. **Price-unit care:** mandi modal price and MSP are quoted **₹ per quintal**, yield in **quintal/acre** — capturing a sale as "₹/kg" risks a 100× error, so the default sale-price unit should be ₹/quintal (kg optional). And the season P&L must use **net** received, not gross: APMC commission, hamali, cess and moisture cut typically take ~6–10% off the gate price.
